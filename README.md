# نَسَق — نظام إدارة المدارس

**نَسَق** هو نظام مكتبي متكامل لإدارة المدارس باللغة العربية، مبني على .NET 8 و WPF و SQL Server. يوفّر إدارة الطلاب والصفوف والحضور والدرجات والرسوم، بالإضافة إلى مركز تقارير وطباعة، ونظام نسخ احتياطي/استعادة، وإدارة مستخدمين وصلاحيات. الواجهة بالكامل عربية مع دعم RTL أصلي وخط Tajawal.

---

## المميزات الرئيسية

- **إدارة الطلاب**: تسجيل كامل بالبيانات الشخصية وأولياء الأمور، نقل بين الصفوف، أرشفة.
- **الصفوف والشعب والمواد**: هيكلة مرنة للمراحل الدراسية والشعب وربط المواد بكل صف.
- **الحضور اليومي**: تسجيل سريع للحضور والغياب والتأخير لكل شعبة بشاشة واحدة.
- **الامتحانات والدرجات**: إنشاء امتحانات، إدخال جماعي للدرجات، احتساب النتائج تلقائياً.
- **النتائج**: عرض نتائج الطلاب لكل صف وشعبة وسنة دراسية مع التقدير وحالة النجاح/الرسوب.
- **الرسوم الدراسية**: خطط رسوم، تعيين للطلاب، تسجيل دفعات، طباعة سندات وكشوف حسابات.
- **مركز التقارير**: تقارير الطلاب، الحضور لفترة، الدرجات، الرسوم — مع تصدير PDF و Excel.
- **الإعدادات العامة**: بيانات المدرسة، الشعار، السنوات الدراسية.
- **المستخدمون والأدوار**: BCrypt لكلمات المرور، أربعة أدوار جاهزة، حماية آخر مدير نظام.
- **النسخ الاحتياطي والاستعادة**: عبر SQL Server مباشرة مع قائمة بيضاء لأسماء قواعد البيانات.
- **مؤشر اتصال قاعدة البيانات**: يظهر في الشريط العلوي ويتحدّث تلقائياً عند انقطاع الاتصال.

## متطلبات التشغيل

| المكوّن | المتطلب |
|--------|---------|
| نظام التشغيل | Windows 10 (1809+) أو Windows 11 |
| إطار العمل | .NET 8 Desktop Runtime |
| قاعدة البيانات | SQL Server 2019 أو أحدث / SQL Server Express / SQL Server LocalDB |
| الذاكرة | 2 GB كحد أدنى (4 GB موصى به) |
| المساحة | 200 MB للبرنامج + مساحة قاعدة البيانات |

> **توصية**: للأجهزة الفردية في المدارس الصغيرة، استخدم **SQL Server LocalDB** — لا يحتاج خادم منفصل ويُثبَّت تلقائياً مع Visual Studio أو يمكن تنزيله مستقلاً.

## التثبيت السريع

1. **ثبّت المتطلبات**:
   - حمّل [.NET 8 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/8.0) وثبّته.
   - حمّل [SQL Server LocalDB](https://learn.microsoft.com/sql/database-engine/configure-windows/sql-server-express-localdb) أو SQL Server Express.
2. **انسخ ملفات البرنامج** إلى مجلد على القرص (مثلاً: `C:\Nasaq`).
3. **شغّل** `Nasag.exe` بصلاحيات مستخدم عادية.
4. **عند أول تشغيل**: تظهر شاشة الترحيب (Splash)، فيقوم البرنامج بـ:
   - الاتصال بقاعدة البيانات (LocalDB افتراضياً).
   - تطبيق تحديثات هيكل القاعدة (Migrations).
   - تحميل البيانات الأولية (Seed Data): الصفوف، المواد، الأدوار، حساب المدير الافتراضي.
5. **إذا تعذّر الاتصال بقاعدة البيانات** يفتح **معالج الإعداد** تلقائياً لاختيار أو إنشاء قاعدة بيانات جديدة.

## معالج إعداد قاعدة البيانات (Setup Wizard)

يظهر المعالج تلقائياً عند فشل الاتصال الأولي بقاعدة البيانات، ويمكن استدعاؤه يدوياً من **الإعدادات → إعادة إعداد قاعدة البيانات**. يتكوّن من خمس خطوات:

1. **الترحيب**: شرح موجز.
2. **اختيار نوع الخادم**: LocalDB، أو SQL Server محلي/شبكي.
3. **بيانات الاتصال**: اسم الخادم، نوع المصادقة (Windows / SQL)، اسم القاعدة.
4. **اختبار الاتصال**: تحقّق من الوصول قبل المتابعة.
5. **إنشاء وتطبيق**: تُنشأ القاعدة (إن لم تكن موجودة) وتُطبَّق التحديثات والبيانات الأولية.

يُحفظ ملف الاتصال المخصّص في:
```
%LOCALAPPDATA%\Nasaq\connection.json
```

## النسخ الاحتياطي والاستعادة

من شاشة **الإعدادات → النسخ الاحتياطي**:

- **نسخة احتياطية**: يُنشأ ملف `.bak` كامل لقاعدة البيانات في المجلد المختار (الافتراضي: `%LOCALAPPDATA%\Nasaq\Backups`).
- **استعادة**: اختر ملف `.bak` ليتم استعادة القاعدة منه.

> **تحذير**: عملية الاستعادة **تستبدل قاعدة البيانات الحالية بالكامل** وتتطلّب **إعادة تشغيل البرنامج**. تأكد من أخذ نسخة احتياطية من الوضع الحالي قبل أي استعادة.

العملية تستخدم أوامر `BACKUP DATABASE` و `RESTORE DATABASE` الأصلية عبر `Microsoft.Data.SqlClient` مع قائمة بيضاء صارمة لأسماء قواعد البيانات منعاً لأي حقن SQL.

## التقارير

يحتوي **مركز التقارير** على أربعة تقارير جاهزة، جميعها بتنسيق A4 أفقي (Landscape):

| التقرير | الفلاتر |
|--------|---------|
| قائمة الطلاب | الصف، الشعبة، الحالة |
| حضور لفترة | من تاريخ إلى تاريخ، الصف، الشعبة |
| كشف الدرجات | الصف، الشعبة، المادة، الامتحان |
| الرسوم والدفعات | الصف، الشعبة، حالة السداد، الفترة |

كل تقرير يدعم:
- **معاينة قبل الطباعة** (FlowDocument).
- **تصدير PDF** عبر QuestPDF.
- **تصدير Excel** عبر ClosedXML.
- **طباعة مباشرة** على أي طابعة مثبتة.

## الصلاحيات والأدوار

أربعة أدوار جاهزة، يمكن تعديل صلاحياتها من **الإعدادات → الأدوار**:

| الدور | الصلاحيات الرئيسية |
|------|---------------------|
| **مدير النظام** | جميع الصلاحيات بما فيها إدارة المستخدمين والنسخ الاحتياطي. |
| **مدير المدرسة** | إدارة الطلاب والصفوف والامتحانات والتقارير، بدون إدارة المستخدمين أو إعدادات النظام. |
| **معلم** | تسجيل الحضور وإدخال الدرجات للصفوف المسندة إليه فقط. |
| **محاسب** | إدارة الرسوم وتسجيل الدفعات وطباعة السندات وتقارير الرسوم. |

تُخفى عناصر القائمة الجانبية تلقائياً حسب صلاحيات المستخدم الحالي.

## استكشاف الأخطاء

| المشكلة | الحل |
|--------|-----|
| **تعذّر الاتصال بقاعدة البيانات** | افتح معالج الإعداد وتأكد من اسم الخادم وبيانات المصادقة، أو شغّل LocalDB يدوياً عبر `sqllocaldb start MSSQLLocalDB`. |
| **فشل تحديث قاعدة البيانات (Migration)** | تأكد أن مستخدم قاعدة البيانات يملك صلاحيات `db_owner` أو `dbcreator`، وأنه لا توجد جلسات مفتوحة على القاعدة. |
| **أيقونة الانقطاع تظهر في الشريط العلوي** | تحقّق من تشغيل خدمة SQL Server (`services.msc` → SQL Server (MSSQLLocalDB) أو الاسم المناسب)، ثم اضغط الأيقونة لإعادة المحاولة. |
| **لا يمكن تسجيل الدخول** | تأكد من حالة الحساب (نشط)، وإذا ضاعت كلمة مرور المدير الوحيد، استعد من نسخة احتياطية سابقة. |
| **شاشة بيضاء عند بدء التشغيل** | راجع سجل الأخطاء في `%LOCALAPPDATA%\Nasaq\logs\`، ثم أعد التشغيل. |

## مسارات البيانات

| المسار | المحتوى |
|-------|---------|
| `%LOCALAPPDATA%\Nasaq\prefs.json` | تفضيلات المستخدم (آخر شعار/سنة دراسية مختارة، إلخ). |
| `%LOCALAPPDATA%\Nasaq\connection.json` | بيانات الاتصال المخصّصة بقاعدة البيانات (إن وُجدت). |
| `%LOCALAPPDATA%\Nasaq\Backups\` | المجلد الافتراضي للنسخ الاحتياطية. |
| `%LOCALAPPDATA%\Nasaq\logs\` | سجلات الأخطاء (إن فعّلت). |

## بنية المشروع (للمطوّرين)

- `Nasag/` — مشروع WPF الرئيسي (Views, ViewModels, Services).
- `Nasag/Data/` — `DbContext`، Migrations، Seeder.
- `Nasag/Themes/` — أنماط (Colors, Buttons, DataGrid, Typography).
- `Nasag/Reports/` — قوالب التقارير وأدوات التصدير.

البناء:
```powershell
cd Nasag
dotnet build
dotnet run
```
